# 文件传输工具

## 技术框架

- 前端使用 React
- 后端使用 NodeJS
- Sqlite
- 杀毒软件使用火绒
- 支持windows\linux
- 生成后通过一个命令行同时启动前后端，可作为windows 服务，并生成 windows 服务的管理脚本
- 使用一条永久Cookie 记录用户的唯一标识（用户ID）

## 项目说明

项目通过三个界面：设定界面，上传界面，下载界面

### 管理界面

1. 进入需要输入管理员密码，密码在后端配置文件配置
2. 设置接受的文件类型，扩展名判断，需要包含常见的类型和扩展名，并支持自定义
3. 是否允许加密压缩文件上传，是否递归检测，是否通过文件内容判断是压缩文件
4. 是否开启上传后调用杀毒软件扫描（进windows 下支持）
5. 上传、下载界面 IP 黑白名单设定，上传下载分别进行配置，支持
6. IP 表达式格式格式，表达式需要举例给用户说明：CIDR前缀表示法、连字符范围表示法、IP地址列表
7. 文件保存路径，默认 D:\FileTransmit\file，如果没有 D 盘自动往后找
8. 如果系统是 linux 文件默认路径在 /var/usr/FileTransmit/下
9.  文件保留时长，默认 24 小时
10. 每半小时自动清理空文件夹，执行自动删除过期的文件，自动删除失效数据库记录

### 上传界面

1. 支持多个文件选择
2. 支持拖拽
3. 重名文件自动后面加序号
4. 提供输入框让用户输入描述
5. 上传后后端返回一个文件Id，随机的 8 位字母，醒目的展示给用户这个ID
6. 服务端用这个id建立文件夹，里面放文件
7. 数据库保存描述和上传用户信息（ip，用户ID，时间，浏览器等详细信息）
8. 用户浏览器保存上传过的ID记录，进入后请求后端，然后自动清理失效记录
9. 提供历史按钮，来查询过去的上传历史
10. 如果开启了杀毒软件，用命令行静默调用杀毒软件

### 下载界面

1. 提供一个输入框，输入文件ID 浏览文件详情

### 文件详情弹窗

1. 显示文件描述和上传信息，下面有文件清单
2. 点击文件可以进行下载
